# Deno Runtime Basics

The foundational Deno reference for lykn projects: security model,
permissions, namespace APIs, Web Platform APIs, configuration, module
resolution, and the standard library. lykn compiles to Deno-native ESM,
so all Deno runtime features apply to compiled output.

This is a condensed reference. For the full treatment of each topic,
see the JS guide `12-deno/12-01-runtime-basics.md`.

Target environment: **Deno**, **ESM-only**, **`deno lint` + `deno fmt`** on compiled
output, lykn/surface syntax throughout.

---

## Security & Permissions

### ID-01: Deno Is Secure by Default

**Strength**: MUST

Deno denies all access unless explicitly granted with permission flags.

```sh
# Good — minimal permissions
deno run --allow-net=api.example.com --allow-read=./data dist/server.js

# Development only
deno run -A dist/server.js
```

### ID-02: Use Granular Permissions

**Strength**: SHOULD

Scope permissions to specific paths, hosts, and variables.

```sh
deno run \
  --allow-read=./config,./data \
  --allow-write=./output \
  --allow-net=api.example.com:443 \
  --allow-env=PORT,DATABASE_URL \
  dist/server.js
```

### ID-03: Permission Prompts for Interactive Use

**Strength**: CONSIDER

Use `--prompt` (default in Deno 2.x) for interactive permission
requests at runtime.

### ID-04: Deny Flags for Exclusion

**Strength**: CONSIDER

`--deny-net=evil.com` blocks specific resources within broader grants.

## Core APIs

### ID-05: File I/O — `Deno:readTextFile` / `Deno:writeTextFile`

```lykn
(bind text (await (Deno:readTextFile "./config.json")))
(await (Deno:writeTextFile "./output.json" (JSON:stringify data)))
```

### ID-06: HTTP Server — `Deno:serve`

```lykn
(Deno:serve (obj :port 8080) (fn (:any req)
  (new Response "Hello from lykn!")))
```

### ID-07: Subprocess — `Deno:Command`

```lykn
(bind cmd (new Deno:Command "lykn"
  (obj :args #a("compile" "src/main.lykn" "-o" "dist/main.js"))))
(bind output (await (cmd:output)))
```

### ID-08: Environment — `Deno:env:get`

```lykn
(bind port (Number (?? (Deno:env:get "PORT") "8080")))
```

### ID-09: CLI Arguments — `Deno:args`

```lykn
(bind args Deno:args)
```

## Module Resolution

### ID-10: ESM Only — Extensions Required

```lykn
(import "./utils.js" (transform))       ;; explicit .js extension
(import "@std/path" (join))              ;; import map specifier
```

### ID-12: `import:meta` for Module Metadata

```lykn
(bind dir import:meta:dirname)
(bind file import:meta:filename)
(bind url import:meta:url)
```

## lykn-Specific: The Compilation Step

### ID-16: lykn → JS → Deno Pipeline

lykn source compiles to JavaScript before Deno runs it.

```sh
# Compile
lykn compile src/main.lykn -o dist/main.js

# Format compiled output
deno fmt dist/

# Run
deno run --allow-net dist/main.js

# Test
deno test test/
```

All Deno permissions, APIs, and configuration apply to the compiled
`.js` output — not to the `.lykn` source files.
