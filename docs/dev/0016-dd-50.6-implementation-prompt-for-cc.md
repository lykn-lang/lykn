# DD-50.6 Implementation Prompt for CC

## Read this first

DD-50.6 design is settled (commit will close the DD draft at
`workbench/dd-50.6-implicit-return-of-statement-fix.md`). The four
design questions resolved as:

- **Q1: A** — `func` without `:returns` whose last body expression is
  statement-only: skip the implicit-return wrapping. Function returns
  `undefined` implicitly.
- **Q2: A** — `func` WITH `:returns` declared whose last body expression
  is statement-only: emit a **compile error**. The user lied about the
  return contract.
- **Q3: C** — **compile-then-check.** Don't classify by surface-form
  head (curated-list approach). Instead, compile the last body
  expression first, then check the result's head against the kernel
  statement-form list. This is more future-proof (catches surface forms
  whose head isn't a statement form but whose compiled kernel form is).
- **Q4: A** — Statement-form list maintained as constants in two
  locations (`STATEMENT_FORM_HEADS` in `forms.rs`, JS-side equivalent
  in `surface.js`), with `compile-both` cross-compiler tests gating
  list drift.

Q3=C is the most architecturally interesting call — it requires
compile-then-check, which means the func-body emit path may need to
compile the last expression twice in some cases (once in expression
context to classify; if statement-typed, again in statement context to
emit the final form). The implementation outline below spells this out.

This is the orthogonal bug surfaced during DD-50.5 iter-3. The
workaround (removing `func` wrapping in tests) lives in
`test/forms/dd-50_test.lykn` iter-3 section. DD-50.6 closes the
underlying bug so future tests don't need to dodge it.

## Deliverable 1 — Rust-side fix (MUST)

**File:** `crates/lykn-lang/src/emitter/forms.rs`.

### 1a. Implement the compile-then-check classification helper

Add a function that determines, given an emitted (kernel) `SExpr`,
whether it's a statement-typed form:

```rust
/// DD-50.6: return true if the emitted form is statement-typed
/// (would not be valid as a `return` argument in JS).
fn is_statement_emitted(expr: &SExpr) -> bool {
    if let SExpr::List { values, .. } = expr
        && let Some(head) = values.first().and_then(|e| e.as_atom())
    {
        return STATEMENT_FORM_HEADS.contains(&head);
    }
    false
}
```

Note: `STATEMENT_FORM_HEADS` already exists from DD-50.5 (around line
2432 of `forms.rs`). Reuse it — don't create a separate list. **Q4=A
means we maintain ONE list in each compiler**, not two parallel lists.

### 1b. Modify the func-body emit paths

For each of the three call sites currently calling
`convert_to_expression` on the last body expression (approx lines
1290, 1318, 1465), apply the following pattern:

```rust
// DD-50.6: compile-then-check the last expression for the
// implicit-return wrapping decision.
let last_emitted = convert_to_expression(last_expr.clone());

if is_statement_emitted(&last_emitted) {
    // Last expression compiled to a statement-typed form.
    // Don't wrap in return.
    if returns_type.is_some() {
        // DD-50.6 Q2=A: compile error — :returns declared but body
        // can't produce a value.
        let head_name = last_emitted
            .as_list()
            .and_then(|v| v.first())
            .and_then(|e| e.as_atom())
            .unwrap_or("<unknown>");
        // Emit a diagnostic via the appropriate mechanism (see
        // pattern from DD-50.5's no-else-if compile-error throw,
        // or push to analysis diagnostics if accessible from here).
        items.push(emit_returns_mismatch_error(
            func_name,
            &returns_type_str,
            head_name,
            last_expr_span,
        ));
    } else {
        // Q1=A: no :returns declared — emit as plain statement,
        // function returns undefined implicitly.
        // Re-compile last_expr in STATEMENT context so it produces
        // valid statement-position JS (vs. the expression-context
        // emit which may have produced a throw or invalid shape).
        let last_as_statement = emit_expr_in_statement_ctx(&last_expr, ctx, registry);
        items.push(last_as_statement);
    }
} else {
    // Expression-typed: wrap in return as usual.
    items.push(list(vec![
        atom("const"),
        atom(&result_var),
        last_emitted,
    ]));
}
```

(Pseudocode — adapt to the actual emit-context idioms.)

**The "re-compile in Statement context" step is the Q3=C compile-twice
behavior.** If the first emit produced a no-else-`if` Rule 2 throw
(because Value/Tail context fired Rule 2), the statement-context
re-emit produces a proper kernel `if` form that codegen renders as a
JS `if` statement.

**MUST verify** by tracing manually: for
`(func test :body (if (= x 1) (console:log "yes")))`:
1. First emit in expression context: `emit_if_expression` fires Rule 2,
   produces `(throw (new TypeError "COMPILE_ERROR: ..."))`.
2. `is_statement_emitted` checks head `throw` → in `STATEMENT_FORM_HEADS`
   → true.
3. `:returns` not declared → fall to else branch.
4. Re-emit in Statement context: kernel-form fallthrough at the
   updated profile uses parent context (Statement); `if` produces a
   kernel `if` form. emit codegen renders as `if (x === 1) console.log("yes")`.
5. Func body emits: `function test() { if (x === 1) console.log("yes"); }`.
   ✓ Valid JS, no return wrap, no Rule 2 throw.

## Deliverable 2 — JS-side fix (MUST)

**File:** `packages/lang/surface.js`.

Find `wrapReturnLast` (or the equivalent function — grep for usages
in the `func` / `fn` macro paths).

Apply symmetric logic to Phase 1:

```js
// DD-50.6: compile-then-check classification.
const STATEMENT_FORM_HEADS = [
  "if", "while", "for", "for-of", "for-in", "do-while", "switch",
  "throw", "return", "break", "continue", "label", "debugger",
  "block", "try", "catch", "finally",
  "var", "const", "let",
  "func", "fn", "class", "type", "export", "import",
];

function isStatementEmitted(emittedForm) {
  if (!Array.isArray(emittedForm) || emittedForm.length === 0) return false;
  const head = emittedForm[0];
  return typeof head === "string" && STATEMENT_FORM_HEADS.includes(head);
}

function wrapReturnLast(bodyExprs, funcName, returnsType, ctx) {
  if (bodyExprs.length === 0) return [];
  const lastExpr = bodyExprs[bodyExprs.length - 1];
  const prefix = bodyExprs.slice(0, -1);

  // Compile-then-check: emit last in expression context first.
  const lastEmittedExpr = compileInExpressionCtx(lastExpr, ctx);

  if (isStatementEmitted(lastEmittedExpr)) {
    if (returnsType) {
      // Q2=A: compile error.
      const head = lastEmittedExpr[0];
      throw new Error(
        `function \`${funcName}\` declared \`:returns ${returnsType}\` but body ends with \`${head}\` ` +
        `(a statement-only form which cannot produce a value). ` +
        `Either: (a) add a return-typed expression after the form, ` +
        `or (b) remove \`:returns ${returnsType}\` from the function declaration.`
      );
    } else {
      // Q1=A: re-compile in statement context, emit as plain statement.
      const lastAsStatement = compileInStatementCtx(lastExpr, ctx);
      return [...prefix.map(e => toStatement(compileExpr(e))), lastAsStatement];
    }
  } else {
    // Expression-typed: wrap in return.
    return [...prefix.map(e => toStatement(compileExpr(e))),
            { type: "ReturnStatement", argument: lastEmittedExpr }];
  }
}
```

be added if they don't already exist as named functions.)
