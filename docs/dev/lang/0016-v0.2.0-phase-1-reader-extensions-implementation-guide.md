## New AST Node Type

One new node type alongside the existing four:

```js
/**
 * @typedef {AtomNode | StringNode | NumberNode | ListNode | ConsNode} AstNode
 *
 * @typedef {Object} AtomNode
 * @property {'atom'} type
 * @property {string} value
 *
 * @typedef {Object} StringNode
 * @property {'string'} type
 * @property {string} value
 *
 * @typedef {Object} NumberNode
 * @property {'number'} type
 * @property {number} value
 * @property {number} [base] - Original radix (2–36) for #NNr literals
 *
 * @typedef {Object} ListNode
 * @property {'list'} type
 * @property {AstNode[]} values
 *
 * @typedef {Object} ConsNode
 * @property {'cons'} type
 * @property {AstNode} car - Element before the dot
 * @property {AstNode} cdr - Element after the dot
 */
```

The `base` property on `NumberNode` is optional. Only present for `#NNr` literals. The compiler uses it to emit native JS literal forms (`0b`, `0o`, `0x`) for bases 2, 8, 16.

## Step 1: Backtick / Comma / Comma-At

### What changes

**In `readAtomOrNumber()` (line 56–57)** — add break characters:

```js
if (ch === ' ' || ch === '\t' || ch === '\n' || ch === '\r' ||
    ch === '(' || ch === ')' || ch === ';' ||
    ch === '`' || ch === ',' || ch === "'") {
  break;
}
```

Without this, input like `` (a `b) `` would consume the backtick as part of atom `a`.

**In `readExpr()`** — add prefix character checks after `(` and `"`, before fallthrough to `readAtomOrNumber()`:

```js
function readExpr() {
  skipWhitespaceAndComments();
  if (pos >= source.length) return null;
  const ch = peek();
  if (ch === '(') return readList();
  if (ch === '"') return readString();

  // Prefix characters
  if (ch === '`') {
    advance();
    const expr = readExpr();
    return { type: 'list', values: [{ type: 'atom', value: 'quasiquote' }, expr] };
  }
  if (ch === "'") {
    advance();
    const expr = readExpr();
    return { type: 'list', values: [{ type: 'atom', value: 'quote' }, expr] };
  }
  if (ch === ',') {
    advance();
    if (peek() === '@') {
      advance();
      const expr = readExpr();
      return { type: 'list', values: [{ type: 'atom', value: 'unquote-splicing' }, expr] };
    }
    const expr = readExpr();
    return { type: 'list', values: [{ type: 'atom', value: 'unquote' }, expr] };
  }

  return readAtomOrNumber();
}
```

### How it works

- `` `expr `` → `(quasiquote expr)` — backtick consumed, next form read and wrapped
- `,expr` → `(unquote expr)` — comma consumed, next form wrapped
- `,@expr` → `(unquote-splicing expr)` — comma and `@` consumed greedily, next form wrapped
- `'expr` → `(quote expr)` — single quote consumed, next form wrapped

### Edge cases

- `` ` `` at EOF: `readExpr()` returns `null`, producing `(quasiquote null)` — expander can reject
- `,@` is greedy: `,` followed by anything other than `@` is plain unquote
- Nested: `` `(a ,b ,@c) `` produces `(quasiquote (a (unquote b) (unquote-splicing c)))` because each `,` triggers when `readExpr()` is called from within `readList()`

## Step 2: `#` Dispatch Router

### New function

```js
/**
 * Read a #-dispatched form.
 * @returns {AstNode | null} Parsed node, or null for comments
 * @throws {Error} For unknown dispatch or parse errors
 */
function readDispatch() {
  advance(); // consume '#'
  if (pos >= source.length) {
    throw new Error('unexpected end of input after #');
  }
  const ch = peek();

  if (ch === ';') return readExprComment();
  if (ch === '|') return readBlockComment();
  if (ch === 'a') return readDataLiteral('array');
  if (ch === 'o') return readDataLiteral('object');
  if (ch === '(') {
    throw new Error('use #a(...) for array literals');
  }
  if (ch >= '0' && ch <= '9') return readRadixLiteral();

  throw new Error(`unknown dispatch character: ${ch}`);
}
```

### Integration in `readExpr()`

Add after the comma handler, before the `readAtomOrNumber()` fallthrough:

```js
if (ch === '#') {
  const node = readDispatch();
  if (node === null) return readExpr(); // comments produce no node; recurse
  return node;
}
```

### Integration in `readList()`

Guard against `null` returns from `readExpr()` (caused by `#;` and `#|...|#` inside lists):

```js
while (pos < source.length && peek() !== ')') {
  const expr = readExpr();
  if (expr !== null) values.push(expr);
  skipWhitespaceAndComments();
}
```

### `#` is NOT a break character

`#` must NOT be added to `readAtomOrNumber()`'s break set. `temp#gen` (DD-11 auto-gensym) must remain a single atom. The `#` dispatch only triggers when `readExpr()` sees `#` as the first character of a new token.

## Step 3: `#;` Expression Comment

### New function

```js
/**
 * Read and discard the next complete form (#; expression comment).
 * @returns {null} No AST node produced
 * @throws {Error} If no form follows #;
 */
function readExprComment() {
  advance(); // consume ';'
  skipWhitespaceAndComments();
  if (pos >= source.length) {
    throw new Error('#; at end of input with no form to discard');
  }
  if (peek() === ')') {
    throw new Error('#; at end of list with no form to discard');
  }
  readExpr(); // read and discard
  return null;
}
```

### Edge cases

| Input | Result |
|-------|--------|
| `#;foo bar` | `[atom 'bar']` |
| `(a #;b c)` | `(a c)` |
| `(a #;b)` | `(a)` |
| `#;#;b c d` → `(a #;#;b c d)` | `(a d)` — each `#;` discards one form |
| `#;(a (b c) d) e` | `[atom 'e']` — discards entire nested structure |
| `#;` at EOF | Error: no form to discard |
| `(a b #;)` | Error: no form to discard |

## Step 4: `#|...|#` Nestable Block Comment

### New function

```js
/**
 * Read and discard a nestable block comment.
 * @returns {null} No AST node produced
 * @throws {Error} If block comment is not closed before EOF
 */
function readBlockComment() {
  advance(); // consume '|' (the '#' was consumed by readDispatch)
  let depth = 1;

  while (pos < source.length && depth > 0) {
    const ch = advance();
    if (ch === '#' && pos < source.length && peek() === '|') {
      advance(); // consume '|'
      depth++;
    } else if (ch === '|' && pos < source.length && peek() === '#') {
      advance(); // consume '#'
      depth--;
    }
  }

  if (depth > 0) {
    throw new Error('unterminated block comment (missing |#)');
  }
  return null;
}
```

### Edge cases

| Input | Result |
|-------|--------|
| `#\| comment \|# foo` | `[atom 'foo']` |
| `#\| outer #\| inner \|# still \|# foo` | `[atom 'foo']` — depth 1→2→1→0 |
| `#\|\|# foo` | `[atom 'foo']` — empty comment |
| `a #\| x \|# b` | `[atom 'a', atom 'b']` |
| `(a #\| x \|# b)` | `(a b)` |
| `#\| unclosed` | Error: unterminated |
| `#\| #\| inner \|#` | Error: outer not closed |

## Step 5: `#a(...)` / `#o(...)` Data Literals

### New function

```js
/**
 * Read a collection data literal (#a or #o).
 * @param {string} formName - 'array' or 'object'
 * @returns {AstNode} List node with formName as head atom
 * @throws {Error} If no opening paren follows
 */
function readDataLiteral(formName) {
  advance(); // consume the letter ('a' or 'o')
  skipWhitespaceAndComments();
  if (pos >= source.length || peek() !== '(') {
    throw new Error(`#${formName[0]} must be followed by (...)`);
  }
  const list = readList(); // reads (...) including parens
  return {
    type: 'list',
    values: [{ type: 'atom', value: formName }, ...list.values],
  };
}
```

### Edge cases

| Input | Result |
|-------|--------|
| `#a()` | `(array)` — empty array |
| `#a(1 2 3)` | `(array 1 2 3)` |
| `#o()` | `(object)` — empty object |
| `#o((name "Duncan") (age 42))` | `(object (name "Duncan") (age 42))` |
| `#o(name (age 42))` | `(object name (age 42))` — shorthand |
| `#a` with no `(` | Error |
| `#(1 2 3)` without letter | Error: "use #a(...) for array literals" |

## Step 6: `#NNr` Radix Literals

### New function

```js
/**
 * Read a radix literal (#NNrVALUE). Base 2–36.
 * @returns {AstNode} Number node with computed value and base
 * @throws {Error} For invalid base, missing r, invalid digits, or missing value
 */
function readRadixLiteral() {
  // Read base digits until 'r'
  let baseStr = '';
  while (pos < source.length && peek() !== 'r') {
    const ch = peek();
    if (ch >= '0' && ch <= '9') {
      baseStr += advance();
    } else {
      throw new Error(`expected 'r' after base in radix literal, got '${ch}'`);
    }
  }

  if (pos >= source.length) {
    throw new Error("expected 'r' after base in radix literal, got end of input");
  }

  advance(); // consume 'r'

  const base = parseInt(baseStr, 10);
  if (base < 2 || base > 36) {
    throw new Error(`radix base must be 2-36, got ${base}`);
  }

  // Read value characters until delimiter
  let valueStr = '';
  while (pos < source.length) {
    const ch = peek();
    if (ch === ' ' || ch === '\t' || ch === '\n' || ch === '\r' ||
        ch === '(' || ch === ')' || ch === ';' ||
        ch === '`' || ch === ',' || ch === "'") {
      break;
    }
    valueStr += advance();
  }

  if (valueStr === '') {
    throw new Error(`missing value after #${baseStr}r`);
  }

  // Validate each digit is legal for the base
  for (const digit of valueStr) {
    const digitVal = parseInt(digit, 36);
    if (Number.isNaN(digitVal) || digitVal >= base) {
      throw new Error(`'${digit}' is not a valid digit in base ${base}`);
    }
  }

  const value = parseInt(valueStr, base);
  return { type: 'number', value, base };
}
```

### The `base` property

Number nodes from `#NNr` carry a `base` property: `{ type: 'number', value: 255, base: 16 }`. This lets the compiler emit native JS literals (`0b`, `0o`, `0x`) for bases 2, 8, 16 and decimal for everything else. Nodes without `base` (regular numbers) continue to emit decimal. This is backwards-compatible.

### Edge cases

| Input | Result |
|-------|--------|
| `#2r11111111` | `{ type: 'number', value: 255, base: 2 }` |
| `#8r377` | `{ type: 'number', value: 255, base: 8 }` |
| `#16rff` | `{ type: 'number', value: 255, base: 16 }` |
| `#16rFF` | `{ type: 'number', value: 255, base: 16 }` — case insensitive |
| `#3r201` | `{ type: 'number', value: 19, base: 3 }` |
| `#36rzz` | `{ type: 'number', value: 1295, base: 36 }` |
| `#0r10` | Error: base must be 2–36 |
| `#1r0` | Error: base must be 2–36 |
| `#37r10` | Error: base must be 2–36 |
| `#2r29` | Error: '9' is not valid in base 2 |
| `#16rGG` | Error: 'G' is not valid in base 16 |
| `#16r` | Error: missing value |
| `(const mask #2r11110000)` | List with num 240 at base 2 |

## Step 7: Dotted-Pair Syntax

### Modified `readList()`

When `.` appears inside a list (as a standalone atom), produce a `cons` node instead of a regular `list` node.

```js
function readList() {
  advance(); // skip '('
  const values = [];
  let sawDot = false;
  let cdrNode = null;

  skipWhitespaceAndComments();
  while (pos < source.length && peek() !== ')') {
    const expr = readExpr();
    if (expr === null) {
      skipWhitespaceAndComments();
      continue;
    }

    if (expr.type === 'atom' && expr.value === '.') {
      if (values.length === 0) {
        throw new Error('dot cannot be first element in list');
      }
      if (sawDot) {
        throw new Error('only one dot allowed per list level');
      }
      sawDot = true;

      // Read the element after the dot
      skipWhitespaceAndComments();
      if (pos >= source.length || peek() === ')') {
        throw new Error('nothing after dot in list');
      }
      cdrNode = readExpr();
      if (cdrNode === null) {
        throw new Error('nothing after dot in list');
      }

      // After reading cdr, next must be ')'
      skipWhitespaceAndComments();
      if (pos < source.length && peek() !== ')') {
        throw new Error('only one element allowed after dot in list');
      }
    } else {
      if (sawDot) {
        throw new Error('only one element allowed after dot in list');
      }
      values.push(expr);
    }
    skipWhitespaceAndComments();
  }
  if (pos < source.length) advance(); // skip ')'

  if (sawDot) {
    if (values.length !== 1) {
      throw new Error('dotted pair must have exactly one element before the dot');
    }
    return { type: 'cons', car: values[0], cdr: cdrNode };
  }

  return { type: 'list', values };
}
```

### Why `.` works as detection

`.` is NOT in `readAtomOrNumber()`'s break characters, so standalone `.` surrounded by whitespace produces atom `{ type: 'atom', value: '.' }`. Atoms like `0.5` contain `.` mid-token and get parsed as numbers by the regex. No break character changes needed for `.`.

### Only `(a . b)` form allowed

Per DD-12's rejection of multiple-head cons patterns: exactly one element before the dot, one after. `(a b . c)` is an error. Users write `(a . (b . c))` for nested destructuring.

### Edge cases

| Input | Result |
|-------|--------|
| `(a . b)` | `{ type: 'cons', car: atom 'a', cdr: atom 'b' }` |
| `(a . (b c))` | `{ type: 'cons', car: atom 'a', cdr: list (b c) }` |
| `(a . (b . c))` | `{ type: 'cons', car: atom 'a', cdr: cons(b, c) }` |
| `(1 . 2)` | `{ type: 'cons', car: num 1, cdr: num 2 }` |
| `(,a . ,b)` | `{ type: 'cons', car: (unquote a), cdr: (unquote b) }` |
| `(. a)` | Error: dot cannot be first |
| `(a .)` | Error: nothing after dot |
| `(a . )` | Error: nothing after dot |
| `(a . b . c)` | Error: only one dot allowed |
| `(a b . c)` | Error: exactly one element before dot |
| `((a . b) (c . d))` | List of two cons nodes — dot is per-list-level |
| `(a b c)` | Normal list — no dot, unchanged behavior |

## Step 8: `#` Mid-Atom Passthrough

**No code changes.** The existing `readAtomOrNumber()` does NOT include `#` in its break characters. The `#` dispatch in `readExpr()` only triggers when `#` is the FIRST character seen by `readExpr()` (start of a new token). Inside an atom being consumed by `readAtomOrNumber()`, `#` is just another character.

This step is test-only to confirm the behavior.

## Integration Points for Phase 2 (Expander)

| Reader output | Expander expectation |
|---|---|
| `(quasiquote expr)` list nodes | Expander applies Bawden's algorithm |
| `(unquote expr)` list nodes | Only valid inside quasiquote — expander validates |
| `(unquote-splicing expr)` list nodes | Only valid inside quasiquote — expander validates |
| `(quote expr)` list nodes | Expander resolves to literal AST construction |
| `{ type: 'cons', car, cdr }` nodes | Expander desugars dotted-pair patterns to `car`/`cdr` access |
| `(array ...)` from `#a(...)` | Transparent — expander/compiler already handle |
| `(object ...)` from `#o(...)` | Transparent — expander/compiler already handle |
| Number nodes with `base` | Transparent to expander; compiler uses `base` for native literal output |
| No nodes from `#;` / `#\|...\|#` | Transparent — nothing to process |
