# DD-25.1 Phase 4: Rust Tests + Cross-Compiler Verification

## Context

Phase 3 extended the Rust AST and classifier for defaults and nesting. Phase 4 adds comprehensive tests and verifies cross-compiler parity with the JS fixtures from Phase 1-2.

---

## Milestone 1: Classifier tests for defaults

**File**: `crates/lykn-lang/src/classifier/forms.rs`, test module

### 1.1 — Object default classification

```lisp
(func f :args ((object :string name (default :number age 0))) :body x)
```
Verify: one clause with one `ParamShape::DestructuredObject` containing two `DestructuredField::Simple` entries. Second entry has `default_value: Some(Number(0.0))`.

### 1.2 — Array default classification

```lisp
(func f :args ((array :number first (default :number second 0))) :body x)
```
Verify: `ArrayParamElement::Typed` with `default_value: Some(Number(0.0))`.

### 1.3 — Multiple defaults

```lisp
(func f :args ((object (default :string name "anon") (default :number age 0))) :body x)
```
Both fields have `default_value: Some(...)`.

### 1.4 — Default error: wrong arity

```lisp
(func f :args ((object (default :number age))) :body x)
```
Error containing "requires".

### 1.5 — Default error: missing type keyword

```lisp
(func f :args ((object (default age 0))) :body x)
```
Error containing "type".

## Milestone 2: Classifier tests for nesting

### 2.1 — Nested object with alias

```lisp
(func f :args ((object :string id (alias :any c (object :string name :string email)))) :body x)
```
Verify: `DestructuredField::Nested` with `alias_name: "c"`, `pattern: DestructuredObject { fields: [Simple("name"), Simple("email")] }`.

### 2.2 — Nested object in array (positional)

```lisp
(func f :args ((array (object :string name) :number score)) :body x)
```
Verify: `ArrayParamElement::Nested { pattern: DestructuredObject { ... } }`, followed by `Typed`.

### 2.3 — Two levels deep

```lisp
(func f :args ((object (alias :any a (object :string city (alias :any g (object :number lat :number lng)))))) :body x)
```
Verify: nested `DestructuredField::Nested` inside another `DestructuredField::Nested`.

### 2.4 — Nesting error: bare object in object without alias

```lisp
(func f :args ((object (object :string name))) :body x)
```
Error containing "must use alias".

### 2.5 — fn with nested params

```lisp
(fn ((object (alias :any c (object :string name)))) x)
```
Verify: `SurfaceForm::Fn` with nested destructured param.

## Milestone 3: Emitter tests for defaults

**File**: `crates/lykn-lang/src/emitter/forms.rs`, test module

### 3.1 — Single-clause func with object default

Construct `SurfaceForm::Func` with a `DestructuredField::Simple(TypedParam { default_value: Some(num(0.0)), ... })`. Verify kernel output contains `(default age 0)` inside the `(object ...)` pattern.

### 3.2 — fn with default

Verify arrow kernel contains `(default name "anon")` in pattern.

### 3.3 — Default survives — type checks still emitted

Verify that both the `(default ...)` kernel form and the `(if ...)` type check appear in output.

## Milestone 4: Emitter tests for nesting

### 4.1 — Single-clause func with nested object

Construct with `DestructuredField::Nested`. Verify kernel output contains `(alias c (object name email))`.

### 4.2 — Array with positional nested

Construct with `ArrayParamElement::Nested`. Verify kernel output contains `(object name)` directly inside `(array ...)`.

### 4.3 — Type checks: only leaf fields

Verify that type checks reference leaf names (`name`, `email`) but NOT the alias name (`c`) when it's `:any`.

## Milestone 5: Overlap detection tests

**File**: `crates/lykn-lang/src/analysis/func_check.rs`

### 5.2 — Defaults don't affect overlap
