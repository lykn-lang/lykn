# DD-25.1 Phase 1: JS Surface Compiler — `default` in Destructured Parameters

## Context

DD-25 established the surface destructuring parser and helper infrastructure. The parser currently recognizes `(default ...)` sub-forms and throws "not yet supported" errors at two locations:
- `parseObjectDestructure` line 255 (type position)
- `parseArrayDestructure` line 340 (list position)

The kernel layer already handles `(default name value)` in both object and array patterns — tested in `test/forms/destructuring-object.test.js` (line 20) and `test/forms/destructuring-array.test.js` (line 24). The JS compiler's `compilePattern` produces `AssignmentPattern` ESTree nodes.

This phase replaces the "not yet supported" errors with actual parsing, and updates the helpers to emit the correct kernel forms.

---

## Milestone 1: Parse `(default :type name value)` in object destructuring

**File**: `src/surface.js`, `parseObjectDestructure` (lines 246-303)

### 1.1 — Replace the rejection with parsing

Current code at line 252-259:
```javascript
if (isArray(cur)) {
    const headVal = cur.values[0];
    const headName = headVal?.type === "atom" ? headVal.value : "";
    if (headName === "default") {
        throw new Error("default values in destructured params are not yet supported ...");
    }
```

Change to:
```javascript
if (headName === "default") {
    // (default :type name value) — 4 elements
    if (cur.values.length !== 4) {
        throw new Error(
            "default in destructured params requires 3 arguments: (default :type name value)",
        );
    }
    const defTypeKw = cur.values[1];
    const defName = cur.values[2];
    const defValue = cur.values[3];
    if (!isKeyword(defTypeKw)) {
        throw new Error("default: first argument must be a type keyword");
    }
    if (defName.type !== "atom") {
        throw new Error("default: second argument must be a parameter name");
    }
    fields.push({
        typeKw: defTypeKw,
        name: defName,
        defaultValue: defValue,
    });
    i += 1; // the list consumed one position
    continue; // must continue from the while loop — currently in an if block
}
```

**Important**: The `(default ...)` list appears at position `i` in the values array (the type-keyword position). It's a single unit that replaces a `:type name` pair — so it consumes 1 position (step 1), not 2.

### 1.2 — Restructure the loop to support `continue`

The current `parseObjectDestructure` uses `if` chains without `continue`. Since `default` needs to `continue` after `i += 1`, restructure the top of the loop to use early-continue pattern:

```javascript
while (i < values.length) {
    const cur = values[i];
    // (default :type name value) sub-form
    if (isArray(cur)) {
        const headVal = cur.values[0];
        const headName = headVal?.type === "atom" ? headVal.value : "";
        if (headName === "default") {
            // ... parse default, push field, i += 1, continue
        }
        // ... other list checks (alias, nested — Phase 2 handles these)
    }
    // :type name pair
    if (isKeyword(cur)) {
        // ... existing logic, i += 2
        continue;
    }
    // ... error cases
}
```

### Verification

```javascript
// Should parse successfully
lykn('(func f :args ((object :string name (default :number age 0))) :body (+ name age))');
// → function f({name, age = 0}) { ... type checks for name (string) and age (number) ... }
```

### 2.2 — Validate: `(rest (default ...))` is nonsensical

After the rest element parsing (line 321), add a check: if someone writes `(rest (default :number x 0))`, that's an error. The current rest parser expects `(rest :type name)` with exactly 3 values, so it'll already fail on arity. But add a targeted error message if rest's second element is a list starting with `default`:

```javascript
if (headName === "rest") {
    if (cur.values.length === 3 && isArray(cur.values[1])) {
        const innerHead = cur.values[1].values[0];
        if (innerHead?.type === "atom" && innerHead.value === "default") {
            throw new Error("rest element cannot have a default value");
        }
    }
    // ... existing rest parsing
}
```

## Milestone 3: Update `paramNameNodes` to emit `(default name value)` kernel forms

**File**: `src/surface.js`, `paramNameNodes` (lines 385-401)

### 3.1 — Object pattern: emit `(default name value)` for defaulted fields

Current:
```javascript
if (p.kind === "object") {
    return [array(sym("object"), ...p.fields.map((f) => f.name))];
}
```

Change to:
```javascript
if (p.kind === "object") {
    const elems = p.fields.map((f) => {
        if (f.defaultValue) {
            return array(sym("default"), f.name, f.defaultValue);
        }
        return f.name;
    });
    return [array(sym("object"), ...elems)];
}
```

This produces the kernel form that the compiler already handles: `(object name (default age 0))`.

### 3.2 — Array pattern: emit `(default name value)` for defaulted fields

Current:
```javascript
if (p.kind === "array") {
    const elems = p.fields.map((f) =>
        f.skip ? sym("_") : f.name,
    );
```

Change to:
```javascript
if (p.kind === "array") {
    const elems = p.fields.map((f) => {
        if (f.skip) return sym("_");
        if (f.defaultValue) return array(sym("default"), f.name, f.defaultValue);
        return f.name;
    });
```

### Verification

```javascript
// Object default
lykn('(func f :args ((object :string name (default :number age 0))) :body name)');
// Should produce: function f({name, age = 0}) { ... }

// Array default
lykn('(func f :args ((array (default :number x 0) (default :number y 0))) :body (+ x y))');
// Should produce: function f([x = 0, y = 0]) { ... }
```

## Milestone 5: Tests

**File**: `test/surface/func-destructuring.test.js` — add new test cases

### 5.1 — Happy path tests

| Test | Source | Expected output assertion |
|------|--------|--------------------------|
| Object with single default | `(func f :args ((object :string name (default :number age 0))) :body ...)` | `{name, age = 0}` in output |
| Object with multiple defaults | `(func f :args ((object (default :string name "anon") (default :number age 0))) :body ...)` | `{name = "anon", age = 0}` |
| Mixed default + non-default | `(func f :args ((object :string name (default :number age 0))) :body ...)` | `{name, age = 0}`, type checks for both |
| Default with `:any` | `(func f :args ((object (default :any name "anon") :number age)) :body ...)` | `{name = "anon", age = 0}`, no type check for `name`, type check for `age` |
| Array with defaults | `(func f :args ((array :number first (default :number second 0))) :body ...)` | `[first, second = 0]` |
| Default value is expression | `(func f :args ((object (default :number x (+ 1 2)))) :body x)` | `{x = 1 + 2}` |
| fn with default | `(bind f (fn ((object :string name (default :number age 0))) (console:log name age)))` | `({name, age = 0}) => { ... }` |
| Default + rest in array | `(func f :args ((array (default :number first 0) (rest :number others))) :body ...)` | `[first = 0, ...others]` |

| rest with default | `(func f :args ((array (rest (default :number x 0)))) :body ...)` | error (arity mismatch on rest) |
