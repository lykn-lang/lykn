# DD-25.1 Phase 2: JS Surface Compiler — Nested Destructuring in Parameters

## Context

DD-25 established the surface destructuring parser. The parser currently rejects nested patterns (lists starting with `object`, `array`, or `alias` in name position or type position) with "not yet supported" errors. The kernel layer already handles nested patterns via `(alias key (object ...))` forms — tested in `test/forms/destructuring-nested.test.js`.

Phase 1 added `default` support. Phase 2 adds nested destructuring. Both use the same parser infrastructure — `parseObjectDestructure` and `parseArrayDestructure`.

---

## Milestone 1: Define the `alias` sub-form syntax and internal representation

### Syntax recap

In object destructuring, nesting MUST use `alias`:
```lisp
(object :string name
        (alias :any addr (object :string city :string zip)))
;; addr is the property key; (object ...) is the nested pattern
```

In array destructuring, nesting is positional — no `alias` needed:
```lisp
(array :number id (object :string name :string email) :number score)
;; The object pattern occupies position 1 in the array
```

### Internal representation

A nested field in the `fields` array:
```javascript
{
    nested: true,
    alias: aliasNameNode,     // the property key (for object) or null (for array)
    typeKw: typeKwNode,       // always the :any keyword (or user-specified)
    pattern: { destructured: true, kind: "object"|"array", fields: [...], ... },
    // name is NOT set — nested fields have multiple bound names via pattern
}
```

For `paramTypeChecks` and `paramBoundNames`, nested fields recurse into the `pattern` to get leaf-level names and type checks.

## Milestone 2: Parse `(alias :type name (object/array ...))` in object destructuring

**File**: `src/surface.js`, `parseObjectDestructure` (lines 246-303)

### 2.1 — Handle `alias` sub-form in type position

Current code at line 260-264 throws "nested destructuring not yet supported" for `alias`. Replace with:

```javascript
if (headName === "alias") {
    // (alias :type alias-name (object/array ...))
    // Minimum 4 elements: alias, type-kw, name, pattern
    if (cur.values.length < 4) {
        throw new Error(
            "alias in destructured params requires: (alias :type name (object/array ...))",
        );
    }
    const aliasTypeKw = cur.values[1];
    const aliasName = cur.values[2];
    const innerPattern = cur.values[3];
    if (!isKeyword(aliasTypeKw)) {
        throw new Error("alias: first argument must be a type keyword");
    }
    if (aliasName.type !== "atom") {
        throw new Error("alias: second argument must be a name");
    }
    if (!isArray(innerPattern)) {
        throw new Error(
            "alias: third argument must be a destructuring pattern (object/array ...)",
        );
    }
    // Recursively parse the inner pattern
    const nestedParam = parseDestructuredParam(innerPattern);
    fields.push({
        nested: true,
        alias: aliasName,
        typeKw: aliasTypeKw,
        pattern: nestedParam,
    });
    i += 1;
    continue;
}
```

### 2.2 — Also handle bare `(object ...)` and `(array ...)` in type position

Currently lines 260-264 throw for `object`/`array` heads too. These represent nested patterns WITHOUT alias, which is an error for object destructuring (no property name):

```javascript
if (headName === "object" || headName === "array") {
    throw new Error(
        "nested pattern in object destructuring must use alias to specify the property name: (alias :type name (object/array ...))",
    );
}
```

### 2.3 — Remove the name-position rejection for nested patterns

Current lines 286-293 reject lists in name position (after a keyword). With `alias` handling in type position (step 1), there's no legitimate case where a nested pattern appears in name position. Keep the error but update the message:

```javascript
if (headName === "object" || headName === "array" || headName === "alias") {
    throw new Error(
        "unexpected nested pattern in name position — use (alias :type name (object/array ...)) as a standalone element",
    );
}
```

### Verification

```javascript
lykn('(func f :args ((object :string id (alias :any customer (object :string name :string email)))) :body (console:log id name email))');
// → function f({id, customer: {name, email}}) { ... type checks for id, name, email ... }
```

Current lines 345-349 reject these. Replace with recursive parsing:

Array elements are positional, so `alias` in array destructuring is less common but could be used for documentation or for the intermediate binding:

```javascript
if (headName === "alias") {

    if (aliasName.type !== "atom") {
        throw new Error("alias: second argument must be a name");
    }
    if (!isArray(innerPattern)) {
        throw new Error(
            "alias: third argument must be a destructuring pattern (object/array ...)",
        );
    }
    const nestedParam = parseDestructuredParam(innerPattern);

        nested: true,
        alias: aliasName,
        typeKw: aliasTypeKw,
```

```javascript

// → function f([{name}, {email}]) { ... }
```

## Milestone 4: Update `paramNameNodes` for nested patterns

**File**: `src/surface.js`, `paramNameNodes` (lines 385-401)

### 4.1 — Object pattern: emit `(alias name (object/array ...))` for nested fields

```javascript
if (p.kind === "object") {
    const elems = p.fields.map((f) => {
        if (f.nested) {
            // Recursively get the inner kernel pattern
            const innerKernel = paramNameNodes(f.pattern)[0];
            if (f.alias) {
                return array(sym("alias"), f.alias, innerKernel);
            }
            // Shouldn't happen for objects (alias required), but defensive
            return innerKernel;
        }
        if (f.defaultValue) {
            return array(sym("default"), f.name, f.defaultValue);
        }
        return f.name;
    });
    return [array(sym("object"), ...elems)];
}
```

### 4.2 — Array pattern: emit nested kernel patterns directly (positional)

```javascript
if (p.kind === "array") {
    const elems = p.fields.map((f) => {
        if (f.skip) return sym("_");
        if (f.nested) {
            const innerKernel = paramNameNodes(f.pattern)[0];
            if (f.alias) {
                // Alias in array context — rare but supported
                return array(sym("alias"), f.alias, innerKernel);
            }
            return innerKernel;
        }
        if (f.defaultValue) return array(sym("default"), f.name, f.defaultValue);
        return f.name;
    });
    if (p.rest) {
        elems.push(array(sym("rest"), p.rest.name));
    }
    return [array(sym("array"), ...elems)];
}
```

### Verification

```javascript
// Object with alias nesting
lykn('(func f :args ((object :string id (alias :any c (object :string name)))) :body (console:log id name))');
// Kernel: (function f ((object id (alias c (object name)))) ...)
// JS: function f({id, c: {name}}) { ... }

// Array with positional nesting
lykn('(func f :args ((array (object :string name) :number score)) :body (console:log name score))');
// Kernel: (function f ((array (object name) score)) ...)
// JS: function f([{name}, score]) { ... }
```

## Milestone 5: Update `paramTypeChecks` for nested patterns

**File**: `src/surface.js`, `paramTypeChecks` (lines 403-419)

### 5.1 — Recurse into nested patterns for leaf-level type checks

```javascript
function paramTypeChecks(p, funcName) {
    if (p.destructured) {
        const checks = [];
        const allFields = [
            ...p.fields.filter((f) => !f.skip),
            ...(p.rest ? [p.rest] : []),
        ];
        for (const f of allFields) {
            if (f.nested) {
                // Recurse into the nested pattern to get leaf-level checks
                checks.push(...paramTypeChecks(f.pattern, funcName));
                // Also type-check the alias binding itself if not :any
                if (f.typeKw && f.typeKw.value !== "any") {
                    const check = buildTypeCheck(f.alias, f.typeKw, funcName, "arg");
                    if (check) checks.push(check);
                }
                continue;
            }
            const check = buildTypeCheck(f.name, f.typeKw, funcName, "arg");
            if (check) checks.push(check);
        }
        return checks;
    }
    const check = buildTypeCheck(p.name, p.typeKw, funcName, "arg");
    return check ? [check] : [];
}
```

**Note**: Per the design doc, intermediate alias names get `:any` (no type check). The inner pattern's leaf fields get their own type checks. If someone writes `(alias :object addr (object :string city))`, the `:object` check on `addr` is emitted too — but typically it's `:any`.

## Milestone 6: Update `paramBoundNames` for nested patterns

**File**: `src/surface.js`, `paramBoundNames` (lines 427-435)

### 6.1 — Include alias names AND recursed leaf names

```javascript
function paramBoundNames(p) {
    if (p.destructured) {
        const names = [];
        for (const f of p.fields) {
            if (f.skip) continue;
            if (f.nested) {
                // Alias name is bound (it's a const binding in the compiled JS)
                if (f.alias) names.push(f.alias);
                // Recurse to get leaf names
                names.push(...paramBoundNames(f.pattern));
                continue;
            }
            names.push(f.name);
        }
        if (p.rest) names.push(p.rest.name);
        return names;
    }
    return [p.name];
}
```

### 7.1 — Happy path tests

| Test | Source | Expected assertion |
|------|--------|--------------------|
| Nested object with alias | `(func f :args ((object :string id (alias :any c (object :string name :string email)))) :body ...)` | `{id, c: {name, email}}`, type checks for id, name, email |
| Nested object in array (positional) | `(func f :args ((array (object :string name) :number score)) :body ...)` | `[{name}, score]`, type checks for name, score |
| Two levels deep | `(func f :args ((object (alias :any a (object :string city (alias :any g (object :number lat :number lng)))))) :body ...)` | `{a: {city, g: {lat, lng}}}` |
| `:any` at intermediate | `(func f :args ((object (alias :any c (object :string name)))) :body ...)` | No type check for `c`, type check for `name` |
| Nested + default combined | `(func f :args ((object (default :string name "anon") (alias :any addr (object :string city)))) :body ...)` | `{name = "anon", addr: {city}}` |
| fn with nested | `(bind f (fn ((object (alias :any c (object :string name)))) (console:log name)))` | `({c: {name}}) => { ... }` |
