## Implementation Order

### 1. Replace `pass0ImportMacros()` stub

Scan for `(import-macros path (bindings...))`. For each:

1. Validate path: string, starts with `./` or `../`, ends `.lykn`
2. Validate binding list present
3. Check for duplicate import of same path
4. Resolve path relative to importing file
5. Check cache (resolved path + mtime via `Deno.statSync()`)
6. Cache miss → `compileMacroModule(resolvedPath, compilationStack)`
7. Extract requested macros from exports, validate they exist
8. Register in `macroEnv` (with `as` renaming if present)
9. Remove `import-macros` forms from output

### 2. `compileMacroModule(path, stack)` — new function

1. Circular check: if `path` in `stack` → error with full cycle
2. Push path onto stack
3. Read: `Deno.readTextFileSync(path)`
4. Parse: `read(source)`
5. Recursive expand: `expandWithContext(forms, path, [...stack])`
6. Track which macros were exported (from `(export (macro ...))` forms)
7. Compile expanded forms to JS: `compile(expandedForms)`
8. Build module-pattern JS: exported macros in return object
9. Execute: `new Function(...MACRO_API_PARAMS, moduleBody)(...MACRO_API_VALUES)`
10. Pop path from stack, cache result
11. Return `{ exports: { name → fn }, path }`

### 3. Export tracking

During Pass 1 of macro modules, `(export (macro name ...))` both registers the macro AND marks it as exported. The module return object includes only exported macros.

### 4. `expand()` context parameter

`expand(forms, { filePath?, compilationStack? })` — optional context for recursive module compilation. Default: no file path, empty stack. Top-level callers unchanged.

### 5. Caching

```js
const moduleCache = new Map(); // resolvedPath → { mtime, exports }
```

`Deno.statSync(path).mtime` compared to cached. Leaf modules only for v0.2.0.

### 6. Shadowing prevention

After Pass 0 registers imported macros, Pass 1 checks: local `(macro name ...)` where `name` already in `macroEnv` from imports → error.

## Error Cases (DD-14)

| Case | Message |
|------|---------|
| File not found | `"macro module not found: ./missing.lykn"` |
| Macro not exported | `"macro 'foo' not exported by ./macros.lykn"` |
| Non-macro requested | `"'format-name' is not a macro in ./utils.lykn"` |
| Circular dependency | `"circular macro module dependency: ./a.lykn → ./b.lykn → ./a.lykn"` |
| Duplicate import path | `"duplicate import-macros for ./macros.lykn"` |
| Shadowing | `"macro 'unless' already defined (imported)"` |
| Missing binding list | `"import-macros requires a binding list"` |
| Invalid path | `"import-macros path must be relative with .lykn extension"` |

## Key Constraints

- **Sync pipeline**: `Deno.readTextFileSync()` + `Deno.statSync()` — no async
- **`$`-prefixed API params**: matching existing `MACRO_API_PARAMS`
- **Recursive three-pass**: each module runs full pass0 → pass1 → pass2
- **No import-all**: explicit binding list required
- **Relative paths only**: `./` or `../`, `.lykn` extension required
