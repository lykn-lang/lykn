# v0.2.0 Phase 4 — Three-Pass Pipeline

## Context

Phases 1–3 are complete (reader extensions, expander core, macro system). The expander already has Pass 1 (iterative fixed-point macro registration) and Pass 2 (full expansion walk) working. Phase 4 completes the three-pass architecture by adding Pass 0 stub (`import-macros`), implementing `macroexpand`/`macroexpand-1` debug utilities, and adding a `deno.json` task for the `--expand` CLI flag.

## What's Already Done (from Phase 3)

- `pass1RegisterMacros()` — iterative fixed-point, dependency detection, circular dep errors
- `pass2ExpandAll()` — full expansion walk with dispatch table
- `expand(forms)` — orchestrates Pass 1 → Pass 2
- Fixed-point per-node macro expansion (limit 1000)
- Duplicate macro name detection
- `macroexpand`/`macroexpand-1` dispatch table entries (currently stub errors)

## What Phase 4 Adds

### 1. Pass 0 Stub: `import-macros` detection

- Add `"import-macros"` to dispatch table with `walk: "import-macros"`
- In `pass0ImportMacros()`: scan forms for `(import-macros ...)`, throw error: `"import-macros not yet implemented (Phase 5)"`
- Update `expand()`: `pass0ImportMacros(forms)` → `pass1RegisterMacros()` → `pass2ExpandAll()`

### 2. `macroexpand` / `macroexpand-1` implementation

- `(macroexpand '(form))` — fully expand the quoted form, print to stderr, erase from output
- `(macroexpand-1 '(form))` — expand one step, print to stderr, erase
- Both require quoted argument (the `'` wraps in `(quote ...)`)
- The handler strips the `quote` wrapper, expands, prints, returns null
- For `macroexpand-1`: only do one macro call (don't re-expand result)

### 3. `--expand` deno task

- Add `deno.json` task: `"expand": "deno eval \"import { read } from './src/reader.js'; import { expand } from './src/expander.js'; const src = Deno.readTextFileSync(Deno.args[0]); console.log(JSON.stringify(expand(read(src)), null, 2));\""` — or better, a small script
