## Module 1: Gensym (`emitter/gensym.rs`)

Deterministic counter for intermediate bindings. NOT using the reader's gensym â€” emitter has its own counter for reproducible output.

```rust
pub struct EmitterGensym {
    counter: usize,
}

impl EmitterGensym {
    pub fn new() -> Self { Self { counter: 0 } }
    pub fn next(&mut self, prefix: &str) -> String {
        let name = format!("{prefix}__gensym{}", self.counter);
        self.counter += 1;
        name
    }
    pub fn reset(&mut self) { self.counter = 0; }
}
```

## Module 3: Type Check Emission (`emitter/type_checks.rs`)

Shared helper producing kernel SExpr for type check assertions.

```rust
pub fn emit_type_check(
    param_name: &str,
    type_keyword: &str,
    func_name: &str,
    label: &str,  // "arg" or "field" or "return"
    span: Span,
) -> Option<SExpr>
```

Returns `None` for `:any`. Otherwise returns:

```
(if <check> (throw (new TypeError "funcName: label 'paramName' expected type, got " + (typeof param))))
```

Type check mapping (identical to JS surface.js `buildTypeCheck`):

| Type | Check (negated) |
|------|-----------------|
| `:number` | `(\|\| (!== (typeof x) "number") (Number:isNaN x))` |
| `:string` | `(!== (typeof x) "string")` |
| `:boolean` | `(!== (typeof x) "boolean")` |
| `:function` | `(!== (typeof x) "function")` |
| `:object` | `(\|\| (!== (typeof x) "object") (=== x null))` |
| `:array` | `(! (Array:isArray x))` |
| `:any` | none |
| user-defined | `(\|\| (!== (typeof x) "object") (\|\| (=== x null) (! (in "tag" x))))` |

## Module 4: Contract Emission (`emitter/contracts.rs`)

```rust
pub fn emit_pre_check(func_name: &str, pre_expr: &SExpr, span: Span) -> SExpr
pub fn emit_post_check(func_name: &str, post_expr: &SExpr, result_var: &str, span: Span) -> SExpr
```

Pre: `(if (! <pre-expr>) (throw (new Error "funcName: pre-condition failed: <sexpr> â€” caller blame")))`

Post: Replace `~` atoms with result var, then:
`(if (! <post-expr>) (throw (new Error "funcName: post-condition failed: <sexpr> â€” callee blame")))`

Uses `diagnostics::serializer::serialize_sexpr()` for the source expression in error messages.

## Module 5: Per-Form Emission Rules (`emitter/forms.rs`)

The main emission logic. Each surface form has an `emit_*` function.

### Simple forms (direct rewrites)

| Form | Emission |
|------|----------|
| `Bind` | `(const name value)` â€” type_ann dropped |
| `Obj` | `(object (key1 val1) (key2 val2) ...)` |
| `Cell` | `(object (value val))` |
| `Express` | `sym("target:value")` |
| `Swap` | `(= target:value (f target:value ...extra))` |
| `Reset` | `(= target:value newval)` |
| `ThreadFirst` | Fold: bare â†’ `(step acc)`, list â†’ `(fn acc ...rest)` |
| `ThreadLast` | Fold: bare â†’ `(step acc)`, list â†’ `(fn ...rest acc)` |
| `KernelPassthrough` | Return raw SExpr unchanged |
| `FunctionCall` | `(head args...)` â€” return as-is list |
| `MacroDef` | Return raw SExpr |
| `ImportMacros` | Return raw SExpr |

### `Type` emission

For each constructor:

- Zero-field: `(const Name (object (tag "Name")))`
- With fields: `(function Name (params...) [type-checks] (return (object (tag "Name") (field1 field1) ...)))`

Wrap in `(block ...)` if multiple constructors.

### `Fn` / `Lambda` emission

`(=> (params...) [type-checks] ...body)`

Parse params, emit type checks (unless stripping), emit body.

### `Func` single clause emission

```
(function name (params...)
  [type-checks]
  [pre-check]
  [if :post: (const _result body) + post-check + (return _result)]
  [else if :returns and not :void: (return body)]
  [else: body])
```

### `Func` multi-clause emission

```
(function name ((rest args))
  (if (&& (=== args:length N) [dispatch-type-checks])
    (block
      (const param0 (get args 0))
      ...
      [full type-checks]
      [pre]
      [body + return]))
  ...more clauses...
  (throw (new TypeError "name: no matching clause for arguments")))
```

Sort clauses: longer arity first, more typed before less typed.

### `Match` emission (always IIFE per Phase 2 decision)

```
((=> ()
  (const target expr)
  (if <pattern1-check> (block [bindings] (return body1)))
  (if <pattern2-check> (block [bindings] (return body2)))
  ...
  [throw if no wildcard default]))
```

Pattern compilation reuses the same logic as JS `compilePattern`:

- Literal â†’ `(=== target literal)`
- Wildcard/binding â†’ no check, bind `(const name target)`
- ADT `(CtorName ...)` â†’ `(=== target:tag "CtorName")`, bind fields from registry
- Obj `(obj :key bind ...)` â†’ type/null/in checks, bind properties
- Guard â†’ nested if inside the pattern's block

### `SomeThreadFirst` / `SomeThreadLast` emission

```
((=> ()
  (const _t0 initial)
  (if (== _t0 null) (return _t0))
  (const _t1 (step1 _t0 ...))  // first or last position
  (if (== _t1 null) (return _t1))
  (return (stepN _tN-1 ...))))
```

### `IfLet` emission (always IIFE)

```
((=> ()
  (const _t expr)
  (if <pattern-check>
    (block [bindings] (return then-body))
    (block (return else-body)))))
```

### `WhenLet` emission (always IIFE)

```
((=> ()
  (const _t expr)
  (if <pattern-check>
    (block [bindings] (return body)))))
```

## Module 6: JSON Serialization (`emitter/json.rs`)

Convert kernel SExpr to `serde_json::Value`:

```rust
pub fn sexpr_to_json(expr: &SExpr) -> serde_json::Value
```

Rules:

- `Atom` â†’ JSON string
- `Keyword` â†’ JSON string (just the value, no `:` prefix)
- `String` â†’ JSON string
- `Number` â†’ JSON number
- `Bool` â†’ JSON boolean
- `Null` â†’ JSON null
- `List` â†’ JSON array (recursively)
- `Cons` â†’ JSON array of [car, ".", cdr]

Top-level output: array of kernel forms â†’ JSON array of arrays.

```rust
pub fn emit_module_json(forms: &[SExpr]) -> String {
    let arr: Vec<serde_json::Value> = forms.iter().map(sexpr_to_json).collect();
    serde_json::to_string_pretty(&arr).unwrap()
}
```

## Awareness Items

| Item | Impact |
|------|--------|
| **serde + serde_json dependencies** | Adds ~3 crates. Standard Rust ecosystem deps. |
| **Emitter gensym separate from reader gensym** | Emitter needs deterministic naming to match JS output. Reader gensym is for expansion. |
| **Always-IIFE for match/if-let/when-let** | Matches Phase 2 JS implementation. Tail context optimization deferred. |
| **Cross-compiler tests need deno** | Tests that diff Rust vs JS output require deno runtime. Skip gracefully if not available. |
| **await handling deferred** | Scan + hoist/async-IIFE logic is complex. Implement basic structure, full await handling in a future phase if time allows. |
